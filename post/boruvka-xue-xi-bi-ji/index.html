Boruvka 学习笔记

2023-08-25 3 min read # 学习笔记

P1 算法介绍

Boruvka 算法是求解最小生成树的算法，其算法流程如下：

  1. 初始每个点为一个连通块。

  2. 每次对于每个连通块 iii，找到最小的边 (u,v)(u,v)(u,v)，满足 u∈i,v∉iu\in i,v\notin iu∈i,v∈/​i。我们设 vvv 所在的连通块为 jjj。

  3. 合并所有的 i,ji,ji,j。

  4. 重复执行 2,3 操作直至只剩 111 个连通块。

正确性证明：

  • 每次加最小边类似于 Kruskal，可以参照其的证明。

  • 若每次 2,3 操作使生成树变成了环，则所有环边一定相等，随便剪去一条边即可。

复杂度证明：每次执行 2,3 操作会使连通块个数减少一半吗，所以时间复杂度为 O(Elog⁡n)O(E\log n)O(Elogn)（其中 EEE 为进行一次操作2的的复杂度）。

Boruvka 在普通的最小生成树问题表现不如 Kruskal，但在完全图/稠密图可能有比 Kruskal/Prim 更佳的表现。

P2 例题

Tree MST

由于求完全图的最小生成树，Kruskal/Prim 都至少是 O(n2)O(n^2)O(n2) 级别的。我们考虑 Boruvka。

考虑如何找出每个点距离其最近的有直接连边的且不在其连通块的点。

“不在其连通块”可以通过维护最近的两个点，要求它们不能在同一个连通块。

具体方法就是可以跑一次树上dp，要求出对于每个点 uuu，求 uuu 的子树中满足上述条件的两个点。

然后跑一次换根dp，把 uuu 的信息下传到 vvv。

时间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)。

Air Reform

由于要求路径上边权最大值最小，考虑建出 Kruskal 重构树。考虑建树的过程，发现深度越大的 lca 点权越小。

考虑求出补图的最小生成树，由于补图边数为 O(n2)O(n^2)O(n2)，不好直接做，考虑 Boruvka 算法。

考虑如何找出每个点距离其最近的在补图中有直接连边的且不在其连通块的点。

发现一个点 vvv 距离点 uuu 越近，代表着在重构树上 lca 点权越小，代表着在重构树上 lca 深度越深，代表着在重构树上的 dfn 值 dfnvdfn_vdfnv​ 与 dfnudfn_udfnu​ 越近。

所以我们只用找到对于每个 uuu，分别找到在重构树上左右两侧 dfn 序与 uuu 的差的绝对值最小的的且在补图中有直接连边的且不在其连通块的点即可。

发现连通块在重构树上的 dfn 序肯定构成一段区间，所以每次直接暴力跳即可，每次直接跳一整个连通块，可以证明其复杂度是对的，为 O(n+m)O(n+m)O(n+m)。

总时间复杂度为 O((n+m)log⁡n)O((n+m)\log n)O((n+m)logn)。
